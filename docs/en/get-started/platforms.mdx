GenieX runs on Snapdragon. Two questions decide your setup:

1. **Where will it run?** Pick the OS / chipset.
2. **What model do you have?** Pick the runtime — `llama_cpp` (any GGUF, NPU/GPU/CPU) or `qairt` (AI Hub bundles, NPU only).

## **1. Pick a platform**

| OS | Chipsets | Interfaces |
|---|---|---|
| **Windows ARM64** *(Compute / Copilot+ PC)* | <ul><li>Snapdragon X Elite</li><li>Snapdragon X2 Elite</li></ul> | <ul><li>[CLI](/en/run/cli/quickstart)</li><li>[Python SDK](/en/run/python/quickstart)</li><li>[Local server](/en/run/cli/local-server)</li></ul> |
| **Android** *(Mobile)* | <ul><li>Snapdragon 8 Elite (`SM8750`)</li><li>Snapdragon 8 Elite Gen 5 (`SM8850`)</li></ul> | <ul><li>[Android SDK](/en/run/android/quickstart) (Kotlin, Maven Central)</li></ul> |
| **Linux ARM64** *(IoT — cameras, robotics, industrial)* | <ul><li>Qualcomm QCS9075</li></ul> | <ul><li>[Native install](/en/run/cli/install)</li><li>[Docker](/en/run/linux/install)</li></ul> |

<Tip>**No device on hand?** Sign in to [Qualcomm Developer Cloud (QDC)](https://qdc.qualcomm.com/) for remote sessions on Snapdragon X, 8 Elite, and Dragonwing IQ-9075. See the [QDC walkthrough in the FAQ](/en/resources/faq#qdc-qualcomm-device-cloud).</Tip>

## **2. Pick a runtime**

| | **llama.cpp** | **QAIRT** |
|---|---|---|
| **Model format** | GGUF (any community model) | AI Hub pre-compiled bundles |
| **Compute units** | NPU / GPU / CPU | NPU only |
| **Quantization picked by** | You (`Q4_0`, `Q8_0`, `F16`, …) | Pre-quantized in the bundle |
| **Best for** | Bringing your own GGUF from Hugging Face | Highest NPU performance on AI Hub models |

Choose **`llama_cpp`** if you want to run any GGUF model from Hugging Face, or need CPU/GPU fallback (e.g. IoT devices without HTP).

Choose **`qairt`** if you want the fastest NPU path and your model is on [Qualcomm AI Hub](https://aihub.qualcomm.com/models/).

### Defaults

If you don't pass a compute unit:

| Runtime | Default compute unit |
|---|---|
| `llama_cpp` | `hybrid` (HTP + CPU per-tensor scheduling — the fast path on Snapdragon) |
| `qairt` | `npu` |

Some model families override this — for example, `gpt-oss` falls back to `npu` on `llama_cpp` because it can't run on the hybrid path.

## **llama.cpp**

The `llama_cpp` runtime executes **any GGUF model** through llama.cpp with Qualcomm's GGML Hexagon backend. Pull any community GGUF from Hugging Face and run it on Snapdragon NPU, Adreno GPU, or pure CPU.

### Compute units

`--device` maps to the underlying hardware as follows:

| Alias | Effect |
|---|---|
| `npu` | Pin to Hexagon NPU (`HTP0`). Best NPU-only path. |
| `gpu` | Adreno GPU via OpenCL. |
| `cpu` | Pure CPU. Forces `nGpuLayers = 0`. |
| `hybrid` *(default)* | Empty `device_id` + `n_gpu_layers=999` — llama.cpp's per-tensor HTP+CPU scheduler. **The fast path on Snapdragon.** |

### Quantization

llama.cpp accepts the full GGML quantization range. **Use `Q4_0` for best Hexagon NPU support** — other quants run on GPU/CPU. See [Quantization & device placement](/en/models/supported#quantization--device-placement).

## **QAIRT**

The `qairt` runtime executes pre-compiled bundles from **Qualcomm AI Hub** through the Qualcomm AI Runtime. NPU-only, with the bundle compiled and quantized for a specific Snapdragon chipset — typically the fastest NPU path when your model is on AI Hub.

### Compute units

QAIRT is **NPU only**.

| Alias | Effect |
|---|---|
| `npu` *(default)* | Pin `HTP0` — the only supported path. |
| `cpu` / `gpu` | Coerced to `npu` with a warning. Never an error. |

### Quantization & context

QAIRT bundles are **pre-quantized at compile time**. The KV cache size and context length are also baked into the bundle — neither can be changed at runtime.

On Android, this means `nGpuLayers != 0` and `nCtx != 0` are rejected with `PARAM_NOT_SUPPORTED`. Leave both at defaults and tune `max_tokens` / `enable_thinking` only.

### Chipset on Android

AI Hub pulls on Android **require an explicit chipset** — auto-detect only runs on Windows on Snapdragon. Use:

| Chipset ID | Chipset |
|---|---|
| `SM8750` | Snapdragon 8 Elite |
| `SM8850` | Snapdragon 8 Elite Gen 5 |

See the [Android API reference](/en/run/android/api-reference#model-manager) for `ModelPullInput.chipset`.

### Limitations

- NPU only — no CPU/GPU fallback.
- Context length and KV cache are fixed by the bundle.
- Adding a new AI Hub model requires registering it on the C++ side first (`third-party/geniex-qairt/models/{llm,vlm}_model_registry.h`).
